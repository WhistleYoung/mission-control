# Mission Control 更新日志

## v1.1.0 (2026-04-06)

### ✨ 新增功能
- **模型用量模块**：新增 Token 用量统计页面，包含：
  - 每日用量趋势
  - 每小时用量趋势（使用北京时间显示）
  - 每月用量汇总（总 tokens 和预估费用）
  - 按 Agent、模型、Provider 多维度统计

### 🐛 问题修复
- **时区显示修复**：使用 `Intl.DateTimeFormat` 配合 `Asia/Shanghai` 时区正确显示北京时间
- **凌晨日期跨天问题**：正确处理 UTC 与北京时间的分界线（北京时间 00:00 = UTC 前一天 16:00）

### ⚡ 性能优化
- 使用标准 IANA 时区名称（`Asia/Shanghai`），兼容开源分发

### 🎨 界面优化
- 模型用量页面采用卡片式布局，支持进度条可视化
- 每月用量顶部显示 3 个月汇总卡片

## v1.0.9 (2026-04-03)

### ✨ 新增功能
- **权限审核模块**：
  - 独立「权限审核」面板（Shield 图标）
  - 待审批标签：展示 exec/plugin 权限请求
  - 历史记录标签：已处理的审批记录持久化存储
  - 支持「运行单次」「运行全部」「拒绝」操作

### 🐛 问题修复
- **模型板块校验**：新增模型时检查同厂家下是否有同ID模型，避免绑定后无法删除
- **Agent名称统一**：创建 `src/lib/agent-config.ts` 共享模块，从 `openclaw.json` 统一读取所有Agent名称
- **日历事件表缺失**：添加缺失的 `calendar_events` 表，避免 events API 500错误
- **侧边栏重复项**：移除重复的「模型」导航项，修复「设置」模块消失问题

### ⚡ 性能优化
- **API缓存机制**：realtime-tasks/cron/settings/approvals API 添加 30 秒缓存，减少重复CLI调用
- **Settings API优化**：clawhub whoami 调用从两次合并为一次，添加 1 分钟缓存

### 🎨 界面优化
- 设置模块移至侧边栏最下方
- 版本号更新为 v1.0.9

## v1.0.8 (2026-04-02)

### ✨ 功能优化
- **实时会话页面优化**：
  - 布局改为左右排版：左侧「后台运行任务」，右侧「实时会话」
  - 新增杀死空闲子进程功能
  - 任务类型标记：「Subagent」(紫色) / 「主进程」(蓝色)
  - 状态指示：绿色脉冲(运行中) / 灰色(空闲) / 红色(失败)
- **系统会话过滤**：自动过滤 cron、heartbeat、openai 系统会话，只显示真正的对话类会话
- **模型编辑功能**：支持编辑模型参数（上下文窗口、最大输出、输入类型、推理模型）
- **模型删除保护**：删除模型前检查是否被 agent 绑定，如有绑定则提示

### 🐛 问题修复
- 修复员工模块模型显示 bug（openclaw.json 优先级问题）
- 修复删除模型时 agents.defaults.models 引用未清理问题
- 修复杀死子进程失败问题（改用 sessions.delete 替代 sessions.abort）

### 🎨 界面优化
- 实时会话页面支持拖拽滚动
- 模型弹窗支持滚动，防止内容过多遮挡
- 版本号更新为 v1.0.8
